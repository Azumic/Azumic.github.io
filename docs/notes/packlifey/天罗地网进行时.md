## Github 流
### 分支管理
- [ ] 将仓库分成一下几个分支：需要写脚本自动化
    ::: file-tree

    - main \# 包涵整合包下的所有文件
      - .git 
      - config \# 配置文件
      - mods \# 模组
      - resourcepacks \# 资源包
      - shaderpacks \# 光影
    - ui \# 主要为 UI 美化、视觉效果的模组、材质包等
      - .git
      - config \# 配置文件
      - mods \# 模组
      - resourcepacks \# 资源包
      - shaderpacks \# 光影
    - core \# 核心模组，整合包的玩法模组
      - .git
      - config \# 配置文件
      - mods \# 模组
    - assist \# 辅助模组，利好游戏体验，比如灵魂出窍、一键砍树等，当然，不是无条件提供的
      - .git
      - config \# 配置文件
      - mods \# 模组
    - docs
      - docs \# 整合包文档

    :::
### 流程要求
每次提交的时候，先标记好每个`文件/目录`属于哪一个分支，然后由 `Action` 自动读取并推送到相应的分支上，并打包源码发送到 `Release` 上，便于后边下载排查问题。

打包均采用 `modrinth` 格式，要求必须含有以下几个部分:  
::: file-tree
- modpack_name.zip
- core.zip
- ui.zip
- assist.zip
:::
